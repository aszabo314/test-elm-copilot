# test-elm-copilot

An Elm counter application that demonstrates GitHub Actions workflow for compiling Elm and deploying to GitHub Pages.

## Project Structure

```
.
├── .github/
│   └── workflows/
│       └── deploy.yml          # GitHub Actions workflow
├── src/
│   └── Main.elm               # Elm application source code
├── index.html                 # Static HTML file that loads elm.js
├── style.css                  # External stylesheet
├── elm.json                   # Elm project configuration
└── elm.js                     # Generated by Elm compiler (created by GitHub Actions)
```

## Features

- ✅ Static `index.html` file (not generated by Elm)
- ✅ External `style.css` stylesheet
- ✅ Elm compiled to `elm.js` output file
- ✅ GitHub Actions workflow for automatic compilation
- ✅ Automatic deployment to GitHub Pages

## Local Development

### Prerequisites

- [Elm 0.19.1](https://guide.elm-lang.org/install/elm.html)

### Compile the Elm App

```bash
elm make src/Main.elm --output=elm.js
```

### View Locally

Open `index.html` in your browser, or use a local web server:

```bash
# Using Python 3
python3 -m http.server 8000

# Or using Node.js http-server
npx http-server
```

Then navigate to `http://localhost:8000`

## GitHub Actions Workflow

The workflow automatically:

1. Checks out the repository
2. Installs Elm 0.19.1
3. Compiles `src/Main.elm` to `elm.js` with optimization
4. Deploys all files to GitHub Pages

The workflow runs on pushes to `main` or `master` branches, and can also be triggered manually.

## GitHub Pages Setup

To enable GitHub Pages for this repository:

1. Go to **Settings** → **Pages**
2. Under **Source**, select **GitHub Actions**
3. The site will be available at: `https://<username>.github.io/<repository>/`
